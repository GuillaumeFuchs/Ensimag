Présentation

Le but de cette page est de regrouper les trucs et astuces pour la programmation en C++/CLI. Cette page contient un résumé des différences de syntaxe entre le C++/CLI et le C++ natif, ainsi qu'un tutoriel montrant comment faire interagir différentes librairies natives et managées. Pour d'autres questions liées au C++/CLI, à Visual Studio et aux librairies qui peuvent être utilisées (libpnl par exemple), on pourra se référer à la page Cpp-CLI-FAQ.

Différences de syntaxe entre du code natif et du code managé

NatifManagé
Déclaration de classe

class ClasseNative{ (...) }

ref class ClasseManagee{ (...) }

Pointeurs (*), handles (^)

ClasseNative *clNat = new ClasseNative();

ClasseManagee ^clMan = gcnew ClasseManagee();

Déréférencement, références, tracking references

  • ClasseNative clNat2 = *clNat;
  • ClasseNative &clNat3 = clNat2;
  • ClasseNative *clNat4 = &clNat2;

  • ClasseManagee clMan2 = *clMan;
  • ClasseManagee %clMan3 = clMan2;
  • ClasseManagee ^clMan4 = %clMan2;
Tableaux

  • int uneDimension[10];
  • int deuxDimensions[10][15];
  • array<int,1> ^uneDimension;
  • array<int,2> ^deuxDimensions;


Un exemple complet de projet

Voici une description pas-à-pas d'une application Windows Form (ou ASP.NET) qui, calcule le prix d'un call vanille à l'aide d'une méthode Monte Carlo.
La partie concernant la gestion des dll natives est un condensé de cette page.

La solution comporte:

  • Un projet C++ natif qui effectue les simulations Monte Carlo et se sert de la librairie libpnl;
  • Un projet C++/CLI pour l'interfaçage .NET-Code natif;
  • Un projet Windows Form pour l'interface graphique.


Attention: ce tutoriel suppose que libpnl a déjà été récupéré et installé sur la machine. (voir la FAQ)

Projet C++ Natif

Création du projet

  1. Depuis Visual Studio, menu Fichier > Nouveau > Projet, sélectionner un projet Win32 depuis l'onglet Visual C++
  2. Donner un nom au projet ("Computations" dans cet exemple) et à la solution ("ProjetTest" dans cet exemple)
  3. Dans l'assistant Application Win32, cliquer sur Suivant, cocher "DLL", "Projet vide" puis cliquer sur Terminer.

Code à insérer

Dans l'explorateur de solution choisissez le dossier Fichiers d'en-tête, ajouter un fichier d'en-tête (.h) ("Computations.h" dans cet exemple), et dans le dossier "Fichiers sources" ajouter un fichier C++ (.cpp) ("Computations.cpp" dans l'exemple).

Ajouter le code suivant au fichier d'en-tête: 

#pragma once
#define DLLEXP   __declspec( dllexport )
namespace Computations{
   DLLEXP void calleuro (double &ic, double &prix, int nb_samples, double T,
              double S0, double K, double sigma, double r); 
}

Et le code suivant au fichier source: 

#include "Computations.h"
#include "iostream"
#include "ctime"
#include "pnl/pnl_random.h"
 
using namespace std;
 
 
void Computations::calleuro (double &ic, double &prix, int nb_samples, double T,
               double S0, double K, double sigma, double r)
{
  double drift = (r - sigma * sigma/2.) * T;
  double sqrt_T = sqrt (T);
  double sum = 0;
  double var = 0;
  PnlRng *rng = pnl_rng_create (PNL_RNG_MERSENNE);
  pnl_rng_sseed (rng, time(NULL));
  double payoff;
  for ( int i=0 ; i<nb_samples ; i++ )
    {
      payoff = S0 * exp (drift + sigma * sqrt_T * pnl_rng_normal (rng));
      payoff = MAX(payoff - K, 0.);
      sum += payoff;
      var += payoff * payoff;
    }
 
  prix = exp (-r*T) * sum / nb_samples;
  var = exp (-2.*r*T) * var / nb_samples - prix * prix;
  ic = 1.96 * sqrt (var / nb_samples); 
  pnl_rng_free (&rng);
}

Linker proprement libpnl au projet (consulter la faq).


Générer le projet. Si tout se passe correctement, le repertoire Debug/ de la solution doit contenir la dll du projet natif (Computations.dll) et un fichier de librairie (Computations.lib).

Projet C++/CLI

Ce projet sert d'interface entre le code natif du projet précédent, et le code managé de l'IHM qui sera définie plus tard.

  1. Ajouter un nouveau projet (bibliothèque de classe en Visual C++) (nommé "Wrapper" dans cet exemple) à la solution
  2. Ajouter le chemin vers le projet natif aux répertoires Include supplémentaires
  3. Ajouter le projet natif comme référence du nouveau projet (Click-droit>References ... > Ajouter une nouvelle référence)
  4. Dans Propriétés -> Editeur de liens -> Entrée, mettre le nom de la dll native dans le champ Chargement différé des DLL (Computations.dll dans cet exemple)

Mettre le code suivant au fichier d'en-tête: 

#pragma once
#include "Computations.h"
using namespace System;
 
namespace Wrapper {
 
	public ref class WrapperClass
	{
	private:
		double intConfiance;
		double prix;
	public:
		WrapperClass() {intConfiance = prix = 0;};
		void getPriceCallEuro(int nbTirages, double T, double S0, double K, double sigma, double r);
		double getPrice() {return prix;};
		double getIC() {return intConfiance;};
	};
}

Remarque: la directive #include "Computations.h" peut être ajoutée au fichier stdafx.h au lieu du fichier d'en-tête.

Ajouter le code suivant au fichier source: 

#include "stdafx.h"
 
#include "Wrapper.h"
 
using namespace Computations;
namespace Wrapper {
	void WrapperClass::getPriceCallEuro(int nbTirages, double T, double S0, double K, double sigma, double r) {
		double ic, px;
		calleuro (ic, px, nbTirages, T, S0, K, sigma, r);
		this->intConfiance = ic;
		this->prix = px;
	}
}


Générer le projet Wrapper.

Projet C#

Il est possible de choisir entre une interface Windows Form et une interface ASP.NET. Les descriptions données ici permettent de lier tous les composants ensemble proprement.

Windows Form

Les descriptions données ici montrent comment linker proprement les différents composants du projet sans faire de recopies inutiles. Une façon plus basique d'exécuter le projet consiste à simplement recopier tout ce qui concerne la partie native du projet (les 3 dll du répertoire 'lib' de libpnl, le fichier 'libpnl.lib' ainsi que le .dll et le .lib du projet natif) dans le répertoire contenant l'exécutable Windows Form. Cette copie peut se faire à la main ou bien dans les évènements liés à la compilation; l'instruction de copie est détaillée dans la partie ASP.NET, ici.


Ajouter à la solution un nouveau projet (projet C#: Windows Form) et le définir comme projet de démarrage (click-droit -> définir comme projet de démarrage).

  1. Ajouter le projet C++/CLI comme référence du nouveau projet
  2. Dans les propriétés du projet, onglet Paramètres, ajouter un nouveau paramètre de type chaîne de caractères (nommé AdditionalPaths dans le code ci-dessous), contenant les chemins d'accès à la dll native et au contenu du répertoire lib de libpnl (format : <chemin1>;<chemin2> )
  3. A partir de la "boite a outils" (onglet en colonne de gauche), ajouter des TextBox pour récupérer les paramètres de l'option et le nombre de tirages Monte Carlo, ainsi que deux Label pour afficher le prix et l'intervalle de confiance ("prixLabel" et "icLabel" dans l'exemple).
  4. Ajouter un bouton (nommé goBtn dans l'exemple)
  5. Ajouter la directive 
    using Wrapper;
    au code source de la forme (Form1.cs dans l'exemple)

Ajouter le code suivant à l'évènement bouton cliqué (explorateur de propriétés, symbole "eclair" pour afficher les noms de méthodes associées aux évènements, double-click sur l'action "click" pour créer et afficher squelette code de reaction au click): 

private void goBtn_Click(object sender, EventArgs e)
{
    // Récupérer les valeurs des paramètres dans les différentes TextBox
    WrapperClass wc = new WrapperClass();
    wc.getPriceCallEuro(nbTirages, maturite, S0, strike, sigma, r);
    prixLabel.Text = wc.getPrice().ToString();
    icLabel.Text = wc.getIC().ToString();
}

Dans le fichier Program.cs, ajouter le code suivant à la place du Main: 

static void Main()
{
    string newPath = string.Concat(Environment.GetEnvironmentVariable("PATH"), ";", Properties.Settings.Default.AdditionalPaths);
    Environment.SetEnvironmentVariable("PATH", newPath, EnvironmentVariableTarget.Process);
    Application.EnableVisualStyles();
    Application.SetCompatibleTextRenderingDefault(false);
    Application.Run(new Form1());
}


Compiler, exécuter.

ASP.NET

Pour un projet ASP.NET, il faut que le serveur IIS sache où récupérer les différentes dll. Il suffit pour ça de les mettre dans un répertoire présent dans le PATH, mais le plus simple est de mettre toutes les dll nécessaires dans le bon répertoire bin.

Ajouter à la solution un nouveau projet (projet C#: Application ASP.NET) et le définir comme projet de démarrage.

  1. Ajouter le projet C++/CLI comme référence du nouveau projet
  2. Ajouter à 'Default.aspx' les textbox et labels nécessaires à la récupération des paramètres et à l'affichage des résultats
  3. Ajouter un bouton permettant de lancer les calculs
  4. Comme pour le Windows Form, ajouter les instructions nécessaires à l'évènement bouton cliqué pour pour lancer les calculs et afficher les résultats


Copie des dll

Dans les évènements post-build du projet natif, copier la dll native vers le répertoire bin du projet ASP.NET. Ainsi, chaque fois que le projet natif sera modifié et recompilé, le bonne version de la dll sera recopiée dans le répertoire bin du projet ASP.NET.
Si la dll native s'appelle TestASP.dll et le projet ASP.NET s'appelle SiteWeb, cela donne: 

copy $(SolutionDir)Debug\TestASP.dll $(SolutionDir)SiteWeb\bin\TestASP.dll

Faire la même chose avec les fichiers .dll et .lib de libpnl.

Dans le fichier Global.asax, il faut spécifier que certaines dll peuvent être présentes dans le répertoire bin du projet ASP.NET. ajouter le code suivant: 

 void Application_Start(object sender, EventArgs e)
 {
     String _path = String.Concat(System.Environment.GetEnvironmentVariable("PATH"), ";", System.AppDomain.CurrentDomain.RelativeSearchPath);
     System.Environment.SetEnvironmentVariable("PATH", _path, EnvironmentVariableTarget.Process);
 }
